# 실행 방법 가이드

## 로컬에서 실행하기

### 방법 1: 브라우저에서 직접 열기 (가장 간단)

1. `index.html` 파일을 찾습니다
2. 파일을 더블클릭하거나 브라우저로 드래그 앤 드롭
3. 브라우저에서 사이트가 열립니다

**주의**: Clipboard API는 HTTPS 환경에서만 작동하므로, 로컬 파일로 열면 클립보드 복사 기능이 작동하지 않을 수 있습니다.

### 방법 2: 로컬 서버 실행 (권장)

#### Python이 설치되어 있는 경우

```bash
# Python 3
python -m http.server 8000

# 또는 Python 2
python -m SimpleHTTPServer 8000
```

그 다음 브라우저에서 `http://localhost:8000` 접속

#### Node.js가 설치되어 있는 경우

```bash
# http-server 설치 (처음 한 번만)
npm install -g http-server

# 서버 실행
http-server -p 8000
```

그 다음 브라우저에서 `http://localhost:8000` 접속

#### VS Code Live Server 확장 프로그램 사용

1. VS Code에서 프로젝트 폴더 열기
2. `index.html` 파일을 우클릭
3. "Open with Live Server" 선택
4. 자동으로 브라우저가 열립니다

### 방법 3: PowerShell에서 간단한 서버 실행 (Windows)

```powershell
# 프로젝트 폴더로 이동
cd D:\Projects\kkk

# Python이 있다면
python -m http.server 8000

# 또는 Node.js가 있다면
npx http-server -p 8000
```

## 사용 방법

1. **데이터 로딩**: 페이지가 열리면 자동으로 구글 시트 데이터를 불러옵니다
2. **초성 검색**: 상단 입력 필드에 초성(ㄱ, ㄴ, ㄷ 등)을 입력하면 실시간으로 검색됩니다
3. **카테고리 필터**: 검색 결과를 카테고리별로 필터링할 수 있습니다
4. **결과 복사**: 검색 결과의 이름을 클릭하면 클립보드에 복사됩니다
5. **검색 히스토리**: 검색한 초성은 자동으로 히스토리에 저장됩니다
6. **정답 등록**: "정답 등록" 버튼을 클릭하여 새 항목을 등록할 수 있습니다 (현재는 UI만 구현됨)

## 문제 해결

### 데이터가 로드되지 않는 경우

- 인터넷 연결을 확인하세요
- 구글 시트 URL이 올바른지 확인하세요
- 브라우저 콘솔(F12)에서 오류 메시지를 확인하세요

### 클립보드 복사가 작동하지 않는 경우

- HTTPS 환경에서 실행해야 합니다 (localhost는 예외)
- 브라우저 권한을 확인하세요
- Chrome, Edge, Firefox 최신 버전을 사용하세요

### CORS 오류가 발생하는 경우

- 구글 시트가 "웹에 게시" 설정이 되어 있는지 확인하세요
- CSV 형식 URL을 사용하고 있는지 확인하세요
